# 美国专利版面陷阱与对策（经验记录）

本文记录在 patents 管线实战中踩到的版面坑及其根治思路，供后续维护与新国别 profile 复用。

---

## L1 — 栏号（column numbers）被当成正文段首

### 现象
转换产物里，正文段落开头冒出孤立数字：`2 lead to an extension…`、`3 due to gradient…`、
`5 Lead 24…`、`7 wound…`、`8 As stated…`；参考页出现 `11 12`；封面区残留孤立 `1.`
并连带使重复的发明标题块未被剥干净。

### 根因
美国专利**不按页编号，按"栏"编号**。每栏顶部（运行页眉 `US X,XXX,XXX B2` 正下方，
实测 y≈68pt）印一个栏号：左栏奇数、右栏偶数。左右两个栏号**印在同一 y**，
取词后被 `group_lines` 并成一行（如 `"1. 2"` / `"3 4"` / `"11 12"`）。

旧的页眉/页脚清洗只认"**单个**纯数字"（`text.isdigit() and len<=4`），而并排栏号
那一行含空格甚至尾点（`1.`），`isdigit()` 判 False → 漏网 → 之后分栏时左号进左栏、
右号进右栏，各自黏到该栏第一句话。封面 `1.` 同理：它是第 1 栏的栏号，挡在重复标题块
前面，导致 `_strip_leading_title_block` 的连续全大写判断从它之后才开始，标题没剥干净。

### 对策：坐标分带 + 内容判定（不是像素裁剪）
不写死像素框去"裁"，而是利用文字层每个词自带的坐标，两道卡死、零误伤：

1. **坐标分带**：只在页面最顶 `header_band_frac`（11%，≈87pt）这条窄带内处理 ——
   栏号恒在此带，正文（y>80）几乎不入带。
2. **内容判定**：带内"**整行只有 1~2 个纯整数（可带尾点）**"的行 = 栏号/页码 → 整行剔除。

实现：把 `reading_order._is_footer_noise` 的判据从"单个纯数字"扩展为整行正则
`^\d{1,4}\.?(?:\s+\d{1,4}\.?)?$`，即可同时吃掉单个页码、左右并排栏号、带尾点栏号。

### 为什么这样稳
- **不误删正文数字**：要被删必须**同时**满足"在顶带"且"整行全是数字"。正文里的
  `claim 1`、`20,000 watts`、`1.5 Tesla`、参考号 `lead 46` 既不在顶带、也都含字母词，
  两条都不满足。Tier0 覆盖校验实测 `missing_chars: 0` 佐证零多删。
- **不写死像素**：带宽按页高比例算，删除按内容判定，对页边距/排版的轻微浮动不敏感，
  对所有美国授权件通用，非单件 hardcode。

### 通用化提示
"运行性版面构件"（栏号、页码、运行页眉、`Sheet N of M`）共性都是**位置规律 + 内容规律**
双重确定。新国别 profile 复用同一思路：先按坐标圈带，再按内容（正则/字典）确认身份后剔除，
不要按固定像素切。

- 修复点：`scripts/pipelines/patents/reading_order.py`（`_is_footer_noise` / `_NUMBER_ROW_RE`）
- 复现件：`Patent_US8364286B2_T1_Shielding`（栏号 1–12 全覆盖各种排布）

## L2 — 首行缩进阈值用错量纲（行高 vs 字宽），导致段落欠分割

### 现象
转换产物里大量段落被并成一大块，只有"明显空行 / 隔页"处才分段；附图说明里每条
`FIG. N …` 也被并进同一段，未各自成行。

### 根因
`_column_paragraphs` 旧判据 `indent_thr = 1.2 * line_h`：用【垂直】行高去当【水平】首行
缩进的阈值，量纲就不对。实测 `line_h=median_height≈15.4pt → 阈值 18.5pt`，而真实首行
缩进仅 ≈9pt、FIG 悬挂缩进 ≈9.4pt、续行抖动 ≈0.5pt。阈值 18.5 远高于缩进信号 9 →
**所有缩进型段落边界都判不出**，只剩"大行距/隔页"还能分段。

### 对策：缩进阈值按"字宽(em)"自适应
首行缩进是水平排版量，典型 ≈1em ≈ 2 个字符宽。改：
`indent_thr = max(2.5, 1.0 * median_char_width(words))`（段间行距 `gap_thr` 是垂直量，
仍按 `line_h` 不变）。字宽随字号/文档自适应，**换 PDF、换排版都稳**，不绑定行高。
实测：FIG 28 条各自成行、正文段落正确切分且不碎、Tier0 `missing_chars=0`。

### 更稳的范式（备选升级，未实现）
单一常数阈值仍是启发式。要更鲁棒可叠加：① **逐文档聚类**——把每栏行首 x 偏移聚成
"0(续行)/~缩进(段首)"两簇自动定界，无魔法常数；② **"上一行短"信号**——两端对齐
正文中只有段末行不顶右边界，下一行即新段。人眼判断换行用的正是 ①②。

### 关联代码
- 修复点：`reading_order.py:_column_paragraphs`（`indent_thr` 改 em 基）
